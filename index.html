执行上下文与作用域

执行上下文


  JavaScript中有三种代码运行环境:Global Code,Function Code,Eval Code.为了表示不同的运行环境,JavaScript中有一个执行上下文.当JavaScript代码执行时,会进入不同的执行上下文,这些执行上下文构成了一个执行上下文栈(Execution Context Stack,ECS).

  当JavaScript代码执行时,第一个进入的总是默认Global Execution Context,所以它总是在ECS的最底部.对于每个Execution Context都有三个重要的属性,变量对象(VariableObject),作用域链(ScopeChain)和this.还可以有其他附加属性.

















Execution Context
VariableObject {vars,function declarations,arguments}
ScopeChain VariableObject+all parent scopes
thisValue ContextObject

VO和AO

VO


  变量对象是与执行上下文相关的数据作用域.存储了在上下文中定义的变量和函数声明.一般VO包含:变量,函数声明,函数形参.




















Global VO
a “Global Var”
foo function
outerFunc function
built-ins

函数表达式不包含在VO之中;没有使用var声明的变量不在VO中.

AO(Activation Object)


  只有全局上下文的变量对象允许通过VO的属性名称间接访问;在函数执行上下文中,VO是不能直接访问的,此时由激活对象(Activation Object)扮演VO的角色.激活对象是在进入函数上下文时刻被创建的,通过函数的arguments属性初始化.
Arguments Object是函数上下文里的激活对象AO中的内部对象,包括callee(指向当前函数的引用),length(真正传递的参数的个数),properties-indexes(函数的参数值)属性.

细看Execution Context


  JavaScript解释器创建Execution Context有两个阶段.
创建阶段(函数被调用，但是开始执行函数内部代码之前)
创建Scope Chain;创建VO/AO;设置this的值.
激活/代码执行阶段
设置变量的值;函数的引用;解释/执行代码.
  创建VO/AO时,JavaScript主要做了以下事情:
根据函数的参数,创建并初始化arguments object;扫描函数内部代码,查找函数声明(对于所有找到的函数声明,将函数名和函数引用存入VO/AO中;如果VO/AO中已经有同名函数,则覆盖);扫描函数内部代码,查找变量声明(对于所有找到的变量声明,将变量名存入VO/AO中，初始化为undefined;如果变量名称跟已经声明的形式参数或函数相同,则变量声明不会干扰已经存在的这类属性).

JavaScript权威指南1

类型,值和变量


  JavaScript原始类型:number,string,boolean,null,undefined.对象类型:object.

算术运算


  JavaScript中的运算符在溢出,下溢或被0整除时不会报错.当数字运算结果超过JavaScript能表示的数字上限,结果是infinity值.下溢是当运算结果无限接近于0并比JavaScript能表示的最小值还小的情况,结果返回0.被0整除在JavaScript并不报错.只返回无穷大(infinity)或负无穷大(-infinity).例外是，0除以0没有意义,结果是非数字值,用NaN表示.不是数字或无法转换为数字的操作数一起使用时返回NaN.NaN与任何值都不相等.用x!=x判断.

二进制浮点和四舍五入错误


  二级制浮点数表示法并不能精确表示类似0.1这样简单的数字.

日期和时间


boolean


  布尔值指代真或假,开或关,是或否.保留字true和false.

null


  null是JavaScript的关键字.用来表述 “空值” .

null是一个特殊的对象值，含义是 “非对象” .
  Boolean是变量的一种取值,表明变量未初始化，如果要查询对象属性或数组元素的值返回undefined则说明这个属性或元素不存在.函数没有返回值,则返回undefined.引用没有提供实参的函数形参的值也会得到undefined.

全局对象


  全局对象的属性是全局定义的符号,JavaScript程序可以直接使用.全局对象的初始属性:
全局属性:比如undefined,infinity,NaN
全局函数:比如isNaN(),parseInt(),eval()
构造函数:比如Date(),RegExp(),String(),Object(),Array()
全局对象:比如Math(),JSON

包装对象


  JavaScript对象是复合值:是属性或已命名值的集合.

  存取字符串,数字或布尔值的属性时创建的临时对象叫包装对象.可通过String(),Number()或Boolean()构造函数显式创建包装对象.

原始值和包装对象”==”, “===”全等运算符视为不等.

不可变的原始值和可变的对象引用


  字符串中的所有方法实际返回的是一个新的字符串值.如果比较两个单独的字符串,当且仅当它们的长度相等且每个索引的字符都相等时,JavaScript才认为它们相等.
  对象的比较并非值的比较:即使两个对象包含同样的属性及相同的值,它们也不相等.各个索引元素完全相等的两个数组也不相等.

  对象通常称为引用类型.对象值都是引用,对象的比较均是引用的比较:当且仅当引用同一个基对象时,它们才相等.

  将对象(或数组)赋值给一个对象,仅仅是赋值的引用值:对象本身并没有被复制.如果想得到一个对象或数组的副本,则必须显式复制对象的每个属性或数组的每个元素.

  比较两个单独的对象或者数组,则必须比较它们的属性或元素.

转换和相等性


  ”=”相等运算符灵活多变.

  一个值转换为另一个值并不意味着两个值相等.

显式类型转换


  做显式转换最简单的方法是使用Boolean(),Number(),String()或Object()函数.当不通过new运算符调用这些函数时，它们会作为类型转换函数按照规则做类型转换.

  除了null或undefined之外的任何值都具有toString()方法,这个方法的执行结果通常和String()方法的返回结果一致.若试图把null或undefined转换为对象,则会抛出类型错误(TypeError).
  JavaScript中的某些运算符会做隐式的类型转换，又是用于类型转换.如果”+”运算符的一个操作符是字符串是字符串,他会将另一个操作符转换为字符串.一元”+”运算符将其操作数转换为数字.

  JavaScript中提供了专门的函数和方法用来做更加精确的数字到字符串(number-to-string)和字符串到数字(string-to-number)的转换.
NUmber类定义的toString()方法可以转进制,默认十进制.

  toFixed()根据小数点后的指定位数将数字转换为字符串,不使用指数记数法.toExponential()使用指数计数法将数字转换为指数形式的字符串,其中小数点前只有一位,小数点后的位数由参数指定.toPrecision()根据指定的有效数字位数将数字转换成字符串.

  parseInt()只能解析整数,parseFloat()则可以解析整数和浮点数.

  parseInt()第二个参数用于指定进制.

对象转换为原始值


  所有的对象都转换为true.

变量作用域


  函数体内，局部变量的优先级高于同名的全局变量.

函数作用域和声明提前


  JavaScript使用函数作用域,没有块级作用域.

  JavaScript的函数作用域是指在函数内声明的所有变量在函数体内始终是可见的，并且变量在声明之前已经可用.JavaScript这个特性被非正式地称为声明提前,即JavaScript函数里声明的所有变量(但不涉及赋值)都被”提前”至函数体的顶部.

  使用var声明变量时,创建的这个属性是不可配置的,也就是说变量无法通过delete运算符删除.

HTML5内敛SVG


  SVG;Scalable Vector Graphics.可伸缩矢量图形.用于网格的基于矢量的图像.使用XML格式定义图像.放大或改变尺寸下质量不会有损失.万维网联盟标准.

Canvas VS SVG


SVG: 使用XML描述2D图形;SVG DOM中的每个元素都可用,可附加JavaScript事件处理器;SVG对象的属性变化，浏览器能自动重现图像.
Canvas: 通过JavaScript绘制2D图像;逐像素渲染;位置发生变化，整个场景需要重新绘制.

JavaScript事件

事件冒泡


  IE中的事件流叫事件冒泡，即事件开始时由最具体的元素接收，然后逐级向上传播到较为不具体的节点。

这个click事件会按照1div 2body 3html 4document顺序传播。所有现代浏览器都支持事件冒泡。

事件捕获


  很少有人使用事件捕获。

DOM事件流


  DOM2级规定的事件流包括事件捕获阶段、处于目标阶段和事件冒泡三阶段.

事件处理程序


  诸如click,load和mouseover，都是事件的名字。响应某个事件的函数就是事件处理程序。事件处理程序名字以”on”开头，因此click事件的事件处理程序就是onclick,load事件处理程序就是onload.
